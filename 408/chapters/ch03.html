📖 必背：二叉树遍历口诀
遍历方式访问顺序记忆口诀
前序遍历根 → 左 → 右先看根，再看左，最后右
中序遍历左 → 根 → 右左根右，中间是根
后序遍历左 → 右 → 根左右根，最后看根
层序遍历从上到下，从左到右用队列，一层层出
📖 必背：BST的查找

从根开始：若key < 当前节点，往左走；若key > 当前节点，往右走。每次排除一半子树，平均查找长度O(log₂n)。

📝 必背算法：二叉树中序遍历（递归）
void inorder(Node* root) { if(!root) return; inorder(root->left); visit(root); inorder(root->right); }
📝 历年真题（2024年·选择题）
一棵二叉树中序遍历序列为 b,c,d,e,f,a,g；层次遍历序列为 a,b,g,c,d,e,f。其后序遍历序列为（ ）。
A.c,e,d,f,b,g,a B.c,e,f,d,b,g,a C.e,f,d,c,b,g,a D.e,g,f,d,b,c,a
💡小白解析：层次遍历第一个为根a；中序中a左侧b,c,d,e,f是左子树，右侧g是右子树。继续用层次序确定每层：第2层有b,g，左侧第一个b是左子树根，g是右子树根。逐步还原整棵树后，后序遍历为"左-右-根"：先遍历左子树全部(得到e,f,d,c,b)，再遍历右子树g，最后根a得到e,f,d,c,b,g,a。选C。

📝 真题 1 | 2017·41题·8分 | 算法设计题
二叉树 T 采用二叉链表存储，设计算法求 T 的高度（层数）。

🎯 出题人考察点（408 高频考点）

① 递归 vs 遍历；② 二叉链表结构；③ 时间空间复杂度

🧭 详细解析（小白逐步版）

**递归思路**：`height(T) = 0`（空）；否则 `= max(height(L), height(R)) + 1`。**非递归**：层序遍历（BFS），每层结束时 +1。

⚠️ 边界（空树返回 0 还是 -1？）未明确；⚠️ 忘判 NULL

📝 真题 2 | 2019·41题·10分 | 算法设计题
二叉树 T 用二叉链表存储，设计算法将 T 中所有结点左右子树交换。

🎯 出题人考察点（408 高频考点）

① 递归遍历；② 指针操作；③ 后序遍历变体

🧭 详细解析（小白逐步版）

**递归**：`swap(T)` 若 T 空返回；否则交换 T->lchild 与 T->rchild，递归 swap 两子树。**本质**：后序遍历框架。

💯 得分点（按点给分）

① 算法思想 (3分)；② 完整代码 (5分)；③ 复杂度 (2分)

⚠️ 小白踩坑预警

⚠️ 交换前没保存原指针；⚠️ 递归终止条件错
