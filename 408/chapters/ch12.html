🎯 核心考点

  • 进程状态转换（就绪→运行→阻塞→就绪）
  • 进程同步与互斥：PV操作、信号量
  • 死锁的四个必要条件与银行家算法
📖 必背：进程三大状态
  • 就绪→运行：CPU调度选中该进程
  • 运行→就绪：时间片用完或被抢占
  • 运行→阻塞：等待I/O或资源
  • 阻塞→就绪：等待的事件完成
📖 必背：PV操作模板

P(S)：wait(S) { S--; if S<0 进程进入阻塞队列 }

V(S)：signal(S) { S++; if S<=0 从阻塞队列唤醒一个进程 }

经典问题：生产者-消费者、读者-写者、哲学家进餐

📖 死锁必要条件

四个条件必须同时满足才会死锁：

  • 互斥：资源一次只给一个进程
  • 占有并等待：拿着资源等别的
  • 非抢占：资源不能强行拿走
  • 循环等待：你等我我等它它等你
📝 历年真题（2023年·PV操作经典题）
有n个进程共享一个文件，规定：
①每次最多允许m(m≤n)个进程同时读文件；
②一个进程在读文件时，其他进程既不能读也不能写；
③一个进程在写文件时，其他进程既不能读也不能写。
请用P、V操作实现进程对文件的读写控制。
💡小白解析：这是"读者-写者"问题的变种，加了一个最大并发数m的限制。设置信号量：
• rw=1（读写互斥）
• limit=m（最大并发控制）
读者进程：P(limit)→P(rw)→读→V(rw)→V(limit)
写者进程：P(limit)→P(rw)→写→V(rw)→V(limit)
这里关键是读写都受limit控制，且读写之间互斥。PV操作题的套路：先识别是什么类型的同步互斥（生产者消费者/读者写者/哲学家进餐），再套用对应模版。
📝 历年真题（2022年·银行家算法）
系统中有5个进程P0-P4和3类资源A、B、C。在T₀时刻的资源分配情况如下表。试问T₀时刻是否安全？（给出安全序列）
已知：A资源总量10，B总量5，C总量7。
当前已分配：P0(0,1,0)、P1(2,0,0)、P2(3,0,2)、P3(2,1,1)、P4(0,0,2)
最大需求：P0(7,5,3)、P1(3,2,2)、P2(9,0,2)、P3(2,2,2)、P4(4,3,3)
💡小白解析：先算Need=Max-Allocation，再算Available=Total-Allocated。Available=(10-7,5-2,7-5)=(3,3,2)。①P1满足Need(1,2,2)≤(3,3,2)→执行后释放，Available=(5,3,2)；②P3满足Need(0,1,1)≤(5,3,2)→执行后Available=(7,4,3)；③P4满足Need(4,3,1)≤(7,4,3)→Available=(7,4,5)；④P0满足Need(7,4,3)≤(7,4,5)→Available=(7,5,5)；⑤P2满足Need(6,0,0)≤(7,5,5)→Available=(10,5,7)。安全序列为：P1→P3→P4→P0→P2。银行家算法步骤：①算Need ②算Available ③找满足条件的进程 ④释放资源 ⑤重复直到所有进程完成或找不到安全序列。
